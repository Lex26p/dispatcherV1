# Dispatcher development log

## Назначение

Этот файл фиксирует ход разработки Dispatcher по шагам.

Журнал нужен, чтобы быстро восстановить:

- что уже сделано;
- в каком порядке выполнялись шаги;
- какие решения были приняты;
- какие проверки выполнялись;
- что делать дальше.

## Рабочие правила

- Работаем в одной ветке: `master`.
- Каждый шаг завершается отдельным commit и push.
- Внутри проекта используем имя `Dispatcher` / `dispatcher`, без `V1`.
- Репозиторий называется `dispatcherV1`, потому что имя `dispatcher` уже занято.
- Используем только `x64-debug` и `x64-release`.
- `x86` не используем.
- `Ninja` не используем.
- Папка `out` является локальной папкой сборки и не хранится в Git.
- `git diff` в рабочих командах не используем.
- В финальном блоке коммита используем `git add .`.
