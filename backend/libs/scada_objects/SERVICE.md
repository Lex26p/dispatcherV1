# scada_objects

## Назначение

`scada_objects` — backend-модуль объектной модели Dispatcher.

Модуль отвечает за универсальное дерево объектов, местоположение оборудования и логическую структуру диспетчеризируемого объекта.

Объектная модель является основой для:

- устройств;
- тегов;
- аварий;
- событий;
- дашбордов;
- мнемосхем;
- прав доступа;
- зон ответственности;
- ППР.

## Границы ответственности

`scada_objects` отвечает за:

- `ObjectNode`;
- `ObjectType`;
- `ObjectTree`;
- правила допустимой иерархии объектов;
- иерархию объектов;
- построение пути объекта;
- поиск объекта по идентификатору;
- получение корневых объектов;
- получение дочерних объектов;
- базовую проверку корректности дерева;
- подготовку связей для будущих устройств и оборудования.

## Что модуль не должен делать

`scada_objects` не должен содержать:

- протокольные драйверы;
- опрос оборудования;
- текущие значения тегов;
- историю тегов;
- аварийную логику;
- команды управления;
- бизнес-логику устройств;
- работу с PostgreSQL напрямую;
- UI-логику.

## Основные типы объектов

Поддерживаются базовые типы:

- `Site`;
- `Building`;
- `Floor`;
- `Room`;
- `Zone`;
- `Cabinet`;
- `Rack`;
- `Line`;
- `EquipmentGroup`;
- `Custom`.

## Базовые правила иерархии

Корневыми объектами могут быть:

- `Site`;
- `Building`;
- `Zone`;
- `Line`;
- `EquipmentGroup`;
- `Custom`.

Примеры допустимой иерархии:

```text
Site
 └── Building
      └── Floor
           └── Room
                └── Cabinet
                     └── Rack
```

```text
Site
 └── Zone
      └── Line
           └── EquipmentGroup
```

```text
Building
 └── Room
      └── Cabinet
```

`Custom` разрешен как гибкий тип для случаев, когда стандартных типов недостаточно.
