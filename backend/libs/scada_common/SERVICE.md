# scada_common

## Назначение

`scada_common` — фундаментальная библиотека общих типов и базовых механизмов Dispatcher.

Этот модуль является самым нижним уровнем backend-архитектуры.

Он не должен содержать бизнес-логику.

Основная задача:

- предоставить единые типы;
- предоставить единые правила обработки ошибок;
- предоставить базовые идентификаторы;
- предоставить общие технические структуры.

---

# Ответственность модуля

Модуль отвечает за:

- версии приложения;
- ошибки;
- результаты выполнения операций;
- идентификаторы сущностей;
- время;
- correlation id;
- базовые технические структуры.

# Почему используются Typed IDs

Нельзя использовать:

```cpp
Uuid id;
```

для всех сущностей.

Проблема:

```cpp
Uuid id;

Device device;

Tag tag;
```

Не видно, какой именно объект идентифицируется.

Используется:

```cpp
DeviceId device_id;

TagId tag_id;
```

Преимущества:

- повышается читаемость;
- уменьшается вероятность ошибок;
- доменная модель становится явной;
- проще сопровождение.
